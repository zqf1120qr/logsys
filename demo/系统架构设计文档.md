# 电商核心业务域微服务架构说明

## 1. 基础服务层 (API 网关层)
- 服务名称: `API-Gateway`
- 描述: 负责统一入口路由，防刷限流。所有外部用户的 HTTP 请求均由该服务接收。
- 路由规则: 将所有请求转发至 `UAA-Router` 服务。

## 2. 路由与认证层 (UAA 路由层)
- 服务名称: `UAA-Router`
- 描述: 负责动态路由分发和请求预处理。
- 依赖调用: 每次请求需通过 RPC 调用 `Security-Auth` 进行 Token 校验。

## 3. 安全与鉴权层 (Security 鉴权认证层)
- 服务名称: `Security-Auth`
- 描述: 负责全链路的安全认证与权限校验。
- 存储依赖: 强依赖缓存组件 `Redis-Cluster` (用于存储 Session 和 Token)。
- 下游调用: 鉴权通过后，将请求放行并调用 `Business-App` 处理实际业务。

## 4. 业务应用层 (Application 业务应用层)
- 服务名称: `Business-App`
- 描述: 负责处理核心订单与用户逻辑。
- 存储依赖: 强依赖关系型数据库 `MySQL-Primary`。
